● How the Skeleton, Auto-refresh, and Manual Refresh work

  ---
  Skeleton

  The skeleton only appears on the very first load — when loading is true. After
  that, every background refresh updates the data silently without any visual
  disruption.

  {loading ? <SkeletonUI /> : <ActualContent />}

  The skeleton components (StatusCardSkeleton, TableRowSkeleton) are built to
  match the exact layout of the real content — same grid, same card dimensions —
  so there's no layout shift when data loads in.

  ---
  Auto-refresh (every 5 minutes)

  const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null);

  useEffect(() => {
    fetchAllReportData();                              // initial load on mount

    intervalRef.current = setInterval(
      () => fetchAllReportData(),
      5 * 60 * 1000                                   // fires every 5 minutes
    );

    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current); // cleanup
    };
  }, [fetchAllReportData]);

  - setInterval starts when the component mounts and calls fetchAllReportData
  every 5 minutes
  - The return () => clearInterval(...) inside useEffect is the cleanup function
  — it runs when the user navigates away from the page, stopping the interval
  and preventing memory leaks
  - useRef holds the interval ID so the cleanup can reference and cancel it

  ---
  Manual Refresh

  const fetchAllReportData = useCallback(async (isManual = false) => {
    if (isManual) setRefreshing(true);
    // ...fetch data...
    if (isManual) setRefreshing(false);
  }, []);

  The same fetchAllReportData function is shared by both auto-refresh and the
  manual button. The isManual flag distinguishes them:

  ┌─────────────────┬────────────────────────┬──────────────────────────────┐
  │                 │   isManual = false     │   isManual = true (button    │
  │                 │         (auto)         │            click)            │
  ├─────────────────┼────────────────────────┼──────────────────────────────┤
  │ Spinner shown   │ No                     │ Yes — button icon spins      │
  ├─────────────────┼────────────────────────┼──────────────────────────────┤
  │ Loading         │ No                     │ No                           │
  │ skeleton        │                        │                              │
  ├─────────────────┼────────────────────────┼──────────────────────────────┤
  │ Data updates    │ Yes                    │ Yes                          │
  └─────────────────┴────────────────────────┴──────────────────────────────┘

  This way the background refresh is completely invisible to the user, while the
  manual button gives clear visual feedback.

  ---
  How to change the interval

  Both Charts.tsx and daily-report/page.tsx have this constant at the top of the
  file:

  const REFRESH_INTERVAL_MS = 5 * 60 * 1000; // 5 minutes

  Just change the multiplier. Examples:

  const REFRESH_INTERVAL_MS = 1 * 60 * 1000;  //  1 minute
  const REFRESH_INTERVAL_MS = 10 * 60 * 1000; // 10 minutes
  const REFRESH_INTERVAL_MS = 30 * 60 * 1000; // 30 minutes

  No other code changes needed — the constant flows through to both setInterval
  and is defined in one place per file.
